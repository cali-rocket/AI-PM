# PM Desk AI

PM Desk AI는 **프로덕트 매니저를 위한 데스크형 멀티 에이전트 워크스페이스**입니다.

이 프로젝트는 하나의 단일 챗봇이 아니라, 서로 다른 역할을 가진 여러 AI 에이전트가 하나의 작업 공간 안에서 협업하는 형태를 지향합니다.

기본 에이전트 구성:

* 서비스 기획 및 아이디에이션 Agent
* 제품 운영 Agent
* 개인 비서 Agent

사용자는 특정 에이전트를 직접 선택해 대화할 수 있고, 각 에이전트는 필요 시 적절한 규칙에 따라 먼저 사용자에게 말을 걸 수 있습니다. 또한 에이전트들은 구조화된 프로토콜을 통해 서로 협업할 수 있습니다.

## 핵심 제품 컨셉

이 프로젝트의 핵심 컨셉은 다음과 같습니다.

**“내 책상 위에 앉아 있는 AI 팀”**

주요 특징:

* 사용자가 원하는 에이전트를 직접 선택해 대화 가능
* 각 에이전트는 서로 다른 역할과 관점을 가짐
* 에이전트는 필요 시 사용자에게 선제적으로 알림/브리핑 가능
* 에이전트끼리 구조화된 방식으로 협업 가능
* 기억은 공유되지만, 해석과 판단은 역할별로 다름

## 초기 범위

MVP는 **읽기 중심(read-first) 어시스턴트**로 시작합니다.

초기 연동 대상:

* Slack (아이디어, 멘션, 주목할 대화)
* Asana (프로젝트 상태, 마감일, blocker)
* Google Calendar (회의, 일정)
* Notion (개인 업무 / 개인 업무 DB)
* Web Search (시장 조사, 경쟁사 업데이트)

초기 MVP에서는 외부 시스템에 대한 자동 쓰기(write)는 기본적으로 허용하지 않습니다.

향후 쓰기 동작 예시:

* Asana task 생성/수정
* Notion 항목 생성/수정
* Slack 메시지 전송

위와 같은 기능은 **명시적 승인 기반**으로만 확장합니다.

## 주요 에이전트

### 1. 서비스 기획 및 아이디에이션 Agent

서비스별 맥락 이해, 아이디어 저장/발전, 시장 조사, 경쟁사 업데이트 추적, 기획 논의를 담당합니다.

### 2. 제품 운영 Agent

Asana 기반 프로젝트 추적, 마감일 관리, 리스크 감지, 운영 브리핑 생성을 담당합니다.

### 3. 개인 비서 Agent

일정 파악, **Notion DB 기반 개인 업무 해석/정리**, 회의 준비, 멘션/주의사항 요약 등 개인 실행 보조를 담당합니다.

## 아키텍처 원칙

* 역할 기반 멀티 에이전트 구조
* 공유 메모리 + 역할별 추론
* 구조화된 에이전트 간 협업
* 사용자에게는 항상 하나의 최종 화자만 응답
* 선제 발화는 제한적이고 우선순위 기반이어야 함
* 읽기 중심으로 시작하고, 쓰기는 나중에
* 외부 도구는 source of truth, 메모리는 구조화된 작업 기억 계층
* 개인 업무의 원본은 Notion DB이며, shared memory의 개인 업무 정보는 요약/참조/캐시로만 사용

이 저장소는 다음 순서로 발전해야 합니다.

1. 제품 컨셉과 시스템 설계를 명확히 정의
2. 에이전트 계약(contracts)과 인터페이스 정의

## 제품 정의

**PM Desk AI는 서비스 전략, 제품 운영, 개인 실행을 하나의 데스크형 멀티 에이전트 워크스페이스 안에서 연결해, PM의 사고·우선순위 판단·추적·실행을 돕는 협업형 AI 시스템이다.**

## 추가 가정 (개인 업무 / Notion DB)

개인 업무 source of truth는 Notion DB이며, MVP에서 사용하는 속성은 아래로 제한합니다.

* 제목 (string)
* 상태 (`not started`, `in progress`, `done`)
* 생성일
* 마감일 (nullable)
* 마지막 수정일

각 Notion DB row는 하나의 페이지로 간주하며, 업무 상세 내용은 해당 페이지 본문(body)에 저장된 것으로 취급합니다.
